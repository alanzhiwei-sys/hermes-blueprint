# Hermes Blueprint — Setup Options

Hermes Blueprint should not lose good customers because the setup feels technical. Use the right setup path for the buyer's confidence level.

## The 3 Setup Paths

| Path | Best For | Buyer Promise | Support Boundary |
|---|---|---|---|
| **DIY Setup** | Technical founders, AI hobbyists, builders who can follow Linux docs | "You can install this yourself and follow the 30-day path." | Docs only; no included troubleshooting beyond supported install path |
| **Guided Setup** | SME owners who want the system but do not want to fight terminal/API setup alone | "Alan installs it with you, verifies it works, and helps you choose your first business win." | One Day 0 call + Day 7 review + stated support window |
| **Done-With-You Setup** | Busy operators who want business workflows, not tooling | "Alan configures the system around your business and helps turn usage into workflows." | Customization scope must be written before work starts |

## Buyer Self-Selection Checklist

Ask the buyer to choose the first statement that feels true.

### Choose DIY if:

- You have used a terminal before
- You can copy/paste commands carefully
- You can create and paste an API key
- You are comfortable reading an error message and checking docs
- You want the lowest-cost route

### Choose Guided Setup if:

- You want Hermes but do not want setup uncertainty
- You are worried about API keys, Linux commands, or config files
- You want someone to verify the install with you
- You want help picking the first business workflow
- You prefer paying to avoid wasted time

### Choose Done-With-You if:

- You have a real business workflow that matters now
- You want custom profile setup
- You want a Day 30 ROI review
- You want Hermes adapted to your industry, not just installed
- Your time is worth more than learning the setup details

## Pre-Setup Confidence Checklist

Before starting, the customer needs:

- [ ] Laptop, mini PC, VPS, or server running Linux / Ubuntu 22.04+
- [ ] Stable internet connection
- [ ] OpenRouter, Anthropic, OpenAI, or DeepSeek API key
- [ ] 30-60 minutes for DIY setup, or 60-90 minutes for guided setup
- [ ] One real business workflow they want Hermes to improve
- [ ] Agreement that Hermes drafts and recommends — it does not send, spend, delete, or publish without approval

## Guided Setup Call Agenda

Use this agenda for Blueprint Pro beta users.

1. **5 min — Fit check**
   - Confirm business type
   - Confirm first workflow problem
   - Confirm buyer understands LLM API costs

2. **20 min — Install + provider setup**
   - Run installer
   - Add API key
   - Run health check
   - Confirm one chat response works

3. **15 min — Personalization interview**
   - Capture services, customers, workflow, tools, tone, boundaries
   - Generate `CUSTOMER_CONTEXT.md`

4. **15 min — First Business Win wizard**
   - Pick one 48-hour business artifact
   - Generate `FIRST_WIN_PLAN.md`
   - Run the first prompt if time allows

5. **5 min — Day 7 review booked**
   - Schedule review
   - Explain success criteria

## Setup Success Criteria

A setup is not successful just because the installer finishes.

A successful setup means:

- [ ] `hermes chat` returns a working answer
- [ ] At least one profile works
- [ ] `CUSTOMER_CONTEXT.md` exists
- [ ] `FIRST_WIN_PLAN.md` exists
- [ ] Customer knows their first 48-hour artifact
- [ ] Customer knows what to do on Day 7

## Sales Language

Use this line for non-technical buyers:

> You do not need to become technical to use Hermes Blueprint. If you can describe your business workflow, Guided Setup gets the system running with you and turns the first session into a usable business artifact.

Use this line for technical buyers:

> If you are comfortable with Linux and API keys, the DIY Kit gives you the full system plus a structured 30-day path.
